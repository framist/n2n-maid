# N2N Maid - 让虚拟局域网像约女仆一样简单 🧹✨

*「那个局域网的黄金时代」*

万物互联口号下真正的连接却显得日益奢侈。人们怀念网吧时代局域网联机的无拘无束，渴望随时随地能像坐在同一张沙发上那样，与远方的好友在这个虚拟世界里并肩作战。然而，横亘在彼此之间的，往往是公网 IP 的缺失、复杂的端口映射以及冷冰冰的防火墙高墙。

现在市面上并非没有成熟的解决方案。Tailscale、ZeroTier 这些工具固然稳定强大，但它们就像带着严格"门禁制度"的高档小区——注册账号、验证身份、授权设备，一个都不能少，对于只想临时组局打一把游戏的伙伴来说，这份仪式感未免太过沉重。

[N2N](https://github.com/ntop/n2n) 恰好解决了这个问题——它像是一把万能钥匙，让你能在互联网上搭建真正"自由"的虚拟局域网。但问题来了：它的命令行配置又让很多人望而却步。

自己喜欢折腾技术，不代表朋友也乐意。如果你扔过去一串 `edge -c mynetwork -l supernode.ntop.org:7777 -u mydevice ...` 的命令，换来的很可能是一句无奈的："太麻烦了，要不算了吧。"

*「把复杂留给后台，把简单还给主人」*

工具的门槛，不应成为友谊的绊脚石。

恩兔酱 (N2N Maid) 就是为此而生的。那些晦涩难懂的参数配置，现在被简化成了一份直观的"家政需求单"——就像预约家政服务一样：
- **去哪里？**(填写 Supernode 地址)
- **暗号是什么？**(输入 Community 名称)  
- **叫什么名字？**(设置设备 ID)

想要组建虚拟局域网？告诉恩兔酱这几项基本信息，以后就能**一键直连**,剩下的技术细节全部由她在后台默默搞定。同时还支持**发现局域网设备、实时测延迟、检查 P2P 打洞状态**等实用功能——让你随时掌握"通道"的健康状况。

*「轻盈、开源、多端、可靠」*

其实市面上并不是没有 N2N 的图形界面工具。在 Windows 生态里，大家可能很熟悉那只著名的“小黄鸭”（EasyN2N）。它功能强大，集成了很多实用工具。但作为一个喜欢在 Linux、macOS、Windows 之间反复横跳的折腾党，我总觉得少了点什么。我们希望，这个工具是[开源透明](https://github.com/framist/n2n-maid)的。因为当一个软件接管了你的网络大门时，信任比什么都重要。这意味着你可以随时检查她的胖次里有没有藏着后门;如果你愿意，甚至可以亲手为她缝制新衣服，或者教她学会新技能。

也得益于 Rust 语言的高性能和 Tauri 框架的轻量化设计，恩兔酱的身材非常苗条。她不会像某些 Electron 应用那样疯狂吞噬内存，也不会在后台偷偷占用过多 CPU。大多数时候，她只是安静地蜷缩在系统托盘的小角落里，默默维持着那条通往你数字世界的秘密通道。
